js笔记，是最很早前初学的积累，有各个小知识点

document.write(123)

  • 其实就相当于隐式调用了toString方法

一切未定义的变量，都会去往上去找，如果全局都没有那就报错，而且内部是可以操作外部的变量

  • var a = “ab”
  • function aa(){ a = 123}
  • aa() 外部的a其实就改变了，因为函数内部根本没有定义，是从外部找的

== 其实是比较两边的字符串形式是否一样 1 == “1”

原始数据类型是可以拷贝clone的，引用数据类型直接拷贝就相当于引用拷贝也就是地址拷贝，无法真正clone的，为浅层拷贝。

  • 引用类型包括Object和Array，Function
  • typeof数组和对象，结果都为”Object”
  • 深度拷贝时，记得判断是数组还是对象，因为要提前定义类型(就是定义接收的类型，如：判断出是数组，就设置target =[]准备接收拷贝内容，最后返回target)，基本上都是递归

判断引用类型可以用toString方法，但要改变内部this，这样就可以返回谁调用的这个方法，一般用call
如，Object.prototype.toString.call([]) –> ‘[object Array]’

字符串和数学相比，会转换为数学相比，字符串和字符串相比，比的是第一位的unicode。如果第一位相同就比第二位。

为什么一般定义方法要在原始类型的原型上定义方法，如Array.prototype…而不在Array上定义方法，猜想:就跟构造函数一样，难道可以在构造函数上定义方法？可以试试。–>函数可以定义属性方法，不过只是定义在constructor上，可以查看’各种好奇’笔记。
有时候不写new也可以，如Array()返回一个[]，这个并不是构造出来的，应该是return出来的。

数组的splice方法，第一个参数为起始位置，第二个参数为截取长度，第三个为插入内容，返回被截取的内容，以数组方式。splice方法是要改变原数组的。

obj.name => obj[‘name’]相当于后面这个，注意是字符串

很多需要重复运算的或者查询的都可以先用一个变量保存下来，如for(var i=0;i<arr.length;i++)这里的arr.length可以先用变量保存起来。

原始值是存在栈里的，没有属性方法(引用类型是存在堆中)，如果想通过原始值访问属性，就会调用包装类，如

  • 访问’abc’.length，js会判断String类型，然后new String(‘abc’)然后进行.length属性查看。这就解释了，对原始值属性操作后进行查看，其实都没变(譬如对length操作)，就是因为在包装类完成的，而非原始值。
  • var num = 123;num.abc = ‘ab’;console.log(num.abc) –> undefined
  • 上面过程调用了两次包装类，添加属性时调用包装类，查找属性时调用包装类——num.abc = ‘ab’时，因为原始值不能定义属性方法，所以系统立即new Number(123) .abc = ‘ab’，因为包装类定义方法属性跟这里面原始值没关系，所以执行完就销毁了—然后要访问num.abc时，系统又会执行包装类—undefined。
  • 上面两次包装类，即使可以同时存在，也不可能是相同的，就跟执行两次new操作后的对象是一样的吗？
  • 包装类，就是将原始值包起来，封装为类

函数执行时默认去执行call方法

  • test() –> test.call()

引用值内容一样也不相等

  • [] == [] –> false
  • {} == {} –> false

    引用值地址一样时就是相等的

  • null === null —> true
  • undefined === undefined —> true

函数行参相当于函数内部var了

  • function ac(ind){
    //var ind
    }

当系统执行js文件前，会通篇扫描一遍，看看有没有语法错误，即使在未执行的函数内，也会报错SyntaxError。

  • function qw(){
    …
    } 即使没执行这个函数，因为里面有语法错误，也会报错。
    其余错误在没执行函数之前，是不会报错的。
  • function qw(){
    adadad()
    } 不会报错，因为还没执行函数。

ES5的严格模式，”use strict”

  • 为什么采取字符串形式？因为要兼容各个浏览器版本，假如要写成函数调用的形式，那么低浏览器内核没兼容就直接执行函数会报错。
  • 这句话，全局上就得写在整个页面的页首，局部函数就得写在函数体内的开始。
  • 严格模式的由来，是因为浏览器是基于es3.0，再加上es5.0的新增方法，那么es3.0和es5.0之间相冲突的部分(es5.0对es3.0很多摒弃的，重写的)，只能按es3.0的方法(因为是基于es3.0的)，如果冲突部分想用es5.0，那么就用严格模式。
  • 严格模式变量必须先声明再使用，如var a=b=123 在严格模式就会报错
    因为未声明。也不能使用es3.0的with方法，用来改变作用域
  • with(这里放对象，作用是往后面包裹的内容加一层作用域链){
    所包裹的内容…被笼罩在with包裹的最近的作用域中
    } with会造成执行缓慢，可想而知，本来正常的作用域链，却要在最里面加一层作用域。

var a =(1,2) –> a = 2

  • 当有括号时，且里面有逗号相隔，则返回括号内最后一个值，不止只有一个逗号相隔可能是(1,2,3)都返回最后一个。

    如果括号内为函数，也是返回最后一个，但是注意，用了括号就相当于函数表达式，就放弃当前函数引用了

  • (function q(){return 1},function w(){return 2}) –> 返回后面一个函数，但是再进行w()就会报错w not a defined，表达式会放弃当前的引用。一般表达式会前面拿一个变量接收。

在if的判断语句中，函数的布尔值默认为true

  • 阻止冒泡，stopPropagation方法，IE9及以下为cancelBubble方法
  • 阻止默认行为，preventDefalut方法

function a(){}

  • a.abc() =…写静态方法
  • a.prototype.abc() =…写实例方法

函数的length属性，得到的是行参个数

(function (a,b){}).length 为2

  • 但是不包含rest参数

    (function (a,…b){}).length 为1
    (function (…a){}).length 为0

  • 也不包括参数默认值，从第一位行参起始到有参数默认值为止才是返回的length

    (function (a,b = 2,c){}).length 为1
    (function (a = 1){}).length 为0
