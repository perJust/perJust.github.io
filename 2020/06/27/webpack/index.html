webpack的部分笔记

  1. webpack相当于将资源以对象形式存储起来，然后进行打包；
  2. bundle[hash:8].js hash:8表示随机产生8位的hash
  3. use:[style-loader, css-loader] ==执行顺序是自右向左，自下而上==，这里的先执行css-loader[表示++解析css模块++],再执行style-loader[将css模块++穿插到style标签内++]
  4. 不在index.html中直接写css模块，譬如：<link src=....。因为webpack会把index.html当成一个模板，然后其他模块通过webpack穿插进去。别再想src里的相对路径，早已改变。
  5. eslint-loader需要在loader第一个执行，可以在options选项里加enforce: 'pre'。previous前置 post后置
webpack的一些选项
  1. modules : a)rules ：exclude,include,test,path b)noParse：/jquery/举例：不去解析jquery里的依赖关系
  2. plugins
  3. externals : 指定的将不打包，externals: {jquery:$}
  4. devtool : 需配置，譬如：devtool:"source-map" 生产环境打包后浏览器点击错误来源 可进行具体查看
  5. watch:true 实时监控
  6. watchOptions: {} 实时监控配置选项 有poll、aggregateTimeout、ignored属性
  7. resolve选项 : modules:[]、mainFields:[]、mainFiles:[]与alias:{}
  8. devServer选项 : 有proxy、port、open(Boolean 默认打开浏览器)、hot(boolean 热更新)
webpack的一些插件
  • 放在plugins: []内

  1. html-webpack-plugin

    操作的：1.引用html模板，当index.html没有时，以已有的任意路径的html为模板.配置template: './src/index.html'，产生的filename: 'index.html'。2. 配置minify进行一些压缩操作。3. 配置chunks:[]选择代码块，譬如：多个js文件，选择性引入某各或某几个js。

  3. clean-webpack-plugin,copy-webpack-plugin
  4. 自带的插件banner-plugin 为打包的JS文件 首部加入注释 可以用来注明一些内容
  5. 自带的插件webpack.DefinePlugin 定义环境变量
  6. webpack.IgnorePlugin : 忽略加载某些模块

  2. file-loader ： 处理JS里的src资源，譬如img.src = './login.png'
