css的笔记

垂直方向的margin共用

  • 上一个节点设置margin-bottom:100px,那么下一个处于它下方的节点设置margin-top:100px就不会有变化，共用margin，且选择里面设置的大的定margin
  • 可通过激活父级BFC，达到非共有margin，但是开发中一般不这样，会影响之前布好的整个结构

display:absolute

  • 会使该元素产生层级，父级如果自适应高度，那么该元素就无法撑开父级元素的高度

body默认的margin为8px

伪元素：可以被css操作，但是没有html里的元素结构

  • 当html里元素标签存在时，就有伪元素，每个元素标签都有伪元素
  • 伪元素天生就是inline的，如果需要设定宽高，要定义block类型
  • 如果要伪元素生效，必须要加content属性

清除浮动，如果利用父级元素的伪元素清除子级浮动，需要定义父级伪元素为block，因为clear：both只能对块级元素清除。

  • 子级浮动，如果触发父级BFC，也可以包裹住子级
  • block，块级元素独占一行，可以设置宽高
  • inline,行内元素，不会独占一行，不可以设置宽高
  • inline-block，行内块级元素，不会独占一行，可以设置宽高

position:absolute
float:left/right

  • 设置了上面两个属性任何一个，都相当于给元素增加了display:inline-block属性

一个行级块元素或者是文本元素，只要里面出现文本了，同一行外面的文本默认就与里面的文本对齐，vertical-align可以改变对其当时，默认为middle

并列选择器

  • div.wrapper 是标签为div且class为wrapper的元素，标签必须写前面
  • #w.wrapper id是w且class为wrapper的元素，前后顺序无要求

    父子选择器

  • div .wrapper div后代含有class为wrapper的元素

css选择器，按照权重大小进行选择，如果权重一样则后面覆盖前面

  • 譬如有些时候，直接写一个class选择器时，无法对这个元素进行样式操作，需要多写几个父子选择器或者自身并列选择器，才可以对元素进行操作 –> 这是因为css权重问题，写父子选择器还是并列选择器，都是为了加权重的。之所以不能进行操作，是因为其他操作该元素的选择器的权重较高。

position默认值为static

overflow:auto 超出的部分可以出现滚动条
  1. overflow:auto；内容会被修剪，超出设置的宽高后会出现滚动条
  2. overflow:scroll;内容会被修剪，不管内容是否超出，都会出现滚动条的位置
  3. overflow:visible;这个是默认值，内容不会被修剪，会出现在元素框外面。
  4. overflow:hidden;内容被修剪，多余的内容被隐藏
  5. overflow:inherit;从父元素那里继承overflow的值。
    横向滚动条： 只要子元素宽高超过父元素且父元素开放了overflow
white-space:nowrap; 只能在同一行显示
