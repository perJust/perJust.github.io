vuex的部分笔记

Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式。它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。

大概理解为：一个数据仓库 共享数据方法

  • ES6的扩展运算符 a={aa:1} b={…a,bb:2} -> b={aa:1,bb:2}
基本操作
  1. 通过Vue.Store实例，部署state(数据状态),还可以部署mutations(方法)…
  2. 从仓库取数据时 $store, 可直接作用在视图插值{ { } }上；
  3. 仓库取出来的数据用计算属性接收，因为每次视图响应 都会更新计算属性 方便取出来的数据实时；
  4. 用vuex的mapState，当一个组件需要获取多个状态时候，将这些状态都声明为计算属性会有些重复和冗余。为了解决这个问题，我们可以使用mapState辅助函数帮助我们生成计算属性，让你少按几次键；
  5. 不能直接改变 store 中的状态。改变 store 中的状态的唯一途径就是显式地提交 (commit) mutation。

mutations 修改状态

  1. 模板通过$store.mutations.commit(‘event’, params),第一个参数是需要提交的事件类型，第二个参数是mutations的符合(即向需要提交的事件内传参)；
  2. 用mapMutations，将仓库的mutations的commit(状态提交)操作，用实例的methods映射；
  3. mutation只能同步，因为每个绑定的状态事件需要在每次commit时实时追踪，一旦Mutation的事件类型commit时内部出现了异步，任何在回调函数中进行的状态的改变都是不可追踪的；可用action
  4. commit(‘’,params) 可以采用

actions
Action 类似于 mutation，不同在于：
  • Action 提交的是 mutation，而不是直接变更状态。
  • Action 可以包含任意异步操作。
    不变更状态，通过异步commit，达到异步提交mutation，使mutation自身追踪状态变更
