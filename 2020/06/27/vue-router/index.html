vue-router简单说明

引入新的vue文件,在路由的index.js中配置

然后在核心的App.vue中 写好router结构

<router-view></router-view>就是将模板的内容挂载到这里

子路由

  1. 创建两个子路由文件.vue 并在父级路由文件加上<router-view></router-view>
  2. 在核心的App.vue里添加Part1子路由的<router-link>,其实也可以在Part1.vue里进行添加(当进入Part1的模板才可以进一步操作查看其子路由文件)
  3. 配置子路由，注意在children内写path时，没有对子路由路径加’/‘
vue-router提供的传参
  1. 通过选项的name传递参数，但是只能传递name参数，实际应用中不常用

    可以在核心app.vue里去显示当前路由后组件的name

    也可以在组件内去显示子组件当前的name

当要显示子组件的name时，父级要避免使用name,可以写在path’/‘(当前页面)

  1. 通过标签中的’to’传参
    标签中的’to’要用v-bind绑定
    基本原理个人理解:’:to’中绑定’name’ 然后路由的时候vue-router会在路由配置里找与之相同的’name’ 进而完成路由
    注意’:to’绑定的写法是对象形式的字符串 “{ }” 传参可以通过params传参 只能使用’params’

app.vue里写好’:to’

对应的组件 进行接收

注意命名的’name’要与路由配置里的一致

可在路由配置文件内传参

调用时也是$route.meta[]获取

mete也可传对象meta:{}
单页面多路由区域
  1. 单页面多路由的意思就是调控一个页面里存在多个<router-view></router-view>时 如何配置
  2. 基本原理就是，在核心app.vue里写多个’router-view’,通过’name’命名(未命名’name’的，默认为default状态)
  3. 路由配置时，将原本的’component:…’改为’components:{}’形式，注意加复数形式
  4. 在<router-view>内可以写样式，最终样式会挂到<template>内所包含的大容器上

    路由配置

    由上可知，首页(‘/‘)时，由于有三个<router-view>且进行了分别的配置，可显示配置的三个组件
    可灵活通过路由显示组件
    如上，当url进入’/re’时，原来’/‘时的左右组件交换了
vue-router通过url传参
  1. 基本原理就是，<router-link to='...'>将需要传参的部署到url上,在路由配置里通过绑定变量来接收传到url上的参数
  2. 注意:定义的传参结构，与路由的接收结构不能缺失，即路由绑定了变量，但没被传参或者传参为空，都将不能正确显示该路由组件
  3. 路由配置接收传参时，可以通过正则判断传过来的是否符合要求(变量后加括号写正则)，不符合的话就不显示该路由配置的组件

redirect重定向

1.重定向的意思就是路由配置后重新指向某个已有的路由指向
2. 在路由配置文件内正常配置，只不过不用写component，因为指向了已有的路由配置，要加一条选项’redirect: …’后面接需要指向的已有的路由
3. 重定向也可以传参

alias别名
  1. 基本原理就是相当于一个组件由多个路径管理；
  2. 跟重定向很相似，但是重定向是将新路径指向原有路径且显示出来的路径没有新路径，显示url是原有路径；
  3. alias是将原有路径基础上，加一条alias选项，写好哪些路径也是指向这个路径下的组件且地址栏显示url新路径;

  1. 基本原理就是在<router-view>外包裹<transtion>；
  2. transition的name必须要填，css
    处理是根据其name操作；
  3. 当处理多个router-view时，可以采用<transition-group>同样要写name，注意：其包裹的子节点必须绑定唯一的key;

路由的mode设置和404处理
  1. 路由的mode设置，即设置默认url的’/#/‘是否消去；
  2. 404的处理就是利用路由配置，处理未做处理的路由进行统一404处理

    路由的钩子函数
    局部某个组件的路由钩子函数
  3. 路由配置中，只有一个钩子函数beforeEnter: function (){};
  4. 在模板中有两个钩子函数可以用beforeRouteEnter: function (){}与beforeRouteLeave: function (){};
  5. 注意在写钩子函数时，不要在核心app.vue里写(因为没作用，所有组件都在里面包含着，没有enter和leave);
  6. 形参三个都要写，因为最后一个形参必须要调用next()才可以正常显示组件；
